TVM编译

TVM编译首先要加载训练好的模型比如TorchScript、HLO或ONNX描述的神经网络模型，对于一个模型，一般关注模型的结构，参数，以及模型的输入shape等信息。

接着需要对推理的数据进行预处理，以达网络模型输入的要求，数据的处理主要包括数据的张量化、数据类型转换为指定格式（如需要的是float16或者int8这些低精度类型），数据预处理完成之后，接着就可以对网络模型进行优化。

TVM 部分主要包括了模型的加载和实时推断的逻辑。以下是涉及 TVM 部署的主要代码部分：

  1. 模型加载:

    • 通过 tvm::runtime::Module::LoadFromFile 从文件加载预训练的 TVM 模型。

  2. 模型初始化和函数获取:

    • 使用模型获取 default 函数，该函数返回一个模型对象。
    • 通过 GetFunction 获取与模型交互的函数，如 set_input、get_output 和 run。

  3. 实时推断循环:

    • 在循环中，通过摄像头捕获图像，将图像数据转换为模型输入格式，设置输入到模型，运行推断，获取模型输出，并对输出应用 softmax 函数。
    • 最后，显示具有最大概率的预测结果。

  4. 摄像头和图像处理:

    • 打开摄像头并进行初始化，处理捕获的图像，将其缩放到模型输入尺寸（28x28）等。

通过以上部分的组合，整个程序实现了使用 TVM 进行实时推断的功能，可以从摄像头捕获图像，输入到预训练的手写数字模型中，然后通过 TVM 进行推断，最终显示出具有最大概率的预测结果。

CMakeLists.txt部分的代码如下

这段代码是一个使用CMake构建的C++项目的CMakeLists.txt文件。以下是对这段代码的构成进行的说明：

  1. CMake最低版本和项目声明: cmake_minimum_required(VERSION 3.20): 指定CMake的最低版本要求为3.20。

project(tvm_demo): 声明项目的名称为 “tvm_demo”。

  1. 设置C++标准:

    设置C++编译标准为C++17。

  2. TVM和DMLC_CORE路径设置:

    设置TVM的根目录为 “/home/cy/tvm/“。

设置DMLC_CORE路径为TVM的dmlc-core子目录。

  1. 设置C++编译标志和包含路径: 添加 -g 编译标志，表示开启调试信息。

使用 include_directories 添加项目所需的头文件路径，包括TVM的include目录、dmlc-core的include目录和dlpack的include目录。

  1. 定义宏:

    使用 add_definitions 定义宏，这里定义了 DMLC_USE_LOGGING_LIBRARY 宏，其值为 <tvm/runtime/logging.h>。

  2. 查找OpenCV库:

    使用 find_package 查找OpenCV库，并将其变量设置为 ${OpenCV_LIBS}。

  3. 生成可执行文件并链接库:

    使用 add_executable 生成可执行文件，其中包括 tvm_mnist.cpp 和注释掉的 tvm_runtime_pack.cc。

使用 target_link_libraries 链接OpenCV库和TVM的运行时库 tvm_runtime 到可执行文件。

可执行文件的源文件是 tvm_mnist.cpp 。最终，该项目将生成一个名为 tvm_demo 的可执行文件。
